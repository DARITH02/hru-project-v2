# សំណើគម្រោង

## ចំណងជើងគម្រោង

**ប្រព័ន្ធគ្រប់គ្រងវត្តមានសិក្សា HRU (HRU-ATMS)**

## 1. សង្ខេបប្រតិបត្តិ

ប្រព័ន្ធគ្រប់គ្រងវត្តមានសិក្សា HRU គឺជាវេទិកាដែលដំណើរការលើបណ្ដាញ ដែលត្រូវបានរចនាឡើងដើម្បីធ្វើទំនើបកម្មការតាមដានវត្តមាន និងការគ្រប់គ្រងរដ្ឋបាលសិក្សានៅក្នុងបរិស្ថានសាកលវិទ្យាល័យ។ ប្រព័ន្ធនេះជំនួសការចុះវត្តមានលើក្រដាស ការត្រួតពិនិត្យកំណត់ត្រាដោយដៃ របាយការណ៍ដែលបែកខ្ញែក និងការតាមដានរដ្ឋបាលដែលយឺតយ៉ាវ ដោយប្រើវេទិកាឌីជីថលមួយដែលមានការប្រមូលផ្ដុំ។

ប្រព័ន្ធដែលបានស្នើនេះគាំទ្រការចូលប្រើតាមតួនាទីសម្រាប់អ្នកគ្រប់គ្រងកំពូល អ្នកគ្រប់គ្រង គ្រូបង្រៀន និងនិស្សិត។ ប្រព័ន្ធរួមបញ្ចូលមុខងារវត្តមានតាម QR code ការគ្រប់គ្រងថ្នាក់សិក្សា វត្តមានគ្រូបង្រៀន ការតាមដានវត្តមាននិស្សិត ការរកឃើញហានិភ័យនៃអវត្តមាន ការគាំទ្របញ្ជីខ្មៅ ការគ្រប់គ្រងពិន្ទុឆមាស ការបង្កើតរបាយការណ៍ PDF និង Excel ការជូនដំណឹងតាម Telegram ការទំនាក់ទំនងពេលវេលាជាក់ស្ដែង និងការដាក់ប្រើប្រាស់ប្រកបដោយសុវត្ថិភាពតាម Docker។

គោលដៅសំខាន់នៃគម្រោងនេះគឺធ្វើឱ្យប្រសើរឡើងនូវភាពត្រឹមត្រូវ ល្បឿន ភាពថ្លាថ្លា និងការទទួលខុសត្រូវក្នុងការគ្រប់គ្រងវត្តមានសិក្សា។ តាមរយៈការផ្លាស់ប្ដូរលំហូរការងារវត្តមានទៅជាឌីជីថល ស្ថាប័នអាចកាត់បន្ថយកំហុសរបស់មនុស្ស សន្សំពេលវេលារដ្ឋបាល កំណត់អត្តសញ្ញាណនិស្សិតដែលមានហានិភ័យបានឆាប់ជាងមុន និងបង្កើតរបាយការណ៍ដែលអាចទុកចិត្តបានសម្រាប់ការសម្រេចចិត្តផ្នែកសិក្សា។

## 2. ផ្ទៃខាងក្រោយ

ស្ថាប័នសិក្សាជាច្រើននៅតែពឹងផ្អែកលើបញ្ជីវត្តមានដោយដៃ សង្ខេបទិន្នន័យតាមសៀវភៅបញ្ជី ឬប្រព័ន្ធដែលមិនភ្ជាប់គ្នា។ វិធីសាស្ត្រទាំងនេះបង្កើតបញ្ហាជាច្រើនដូចខាងក្រោម៖

- កំណត់ត្រាវត្តមានអាចបាត់បង់ ស្ទួន ឬបញ្ចូលខុស។
- គ្រូបង្រៀនចំណាយពេលបន្ថែមក្នុងការត្រួតពិនិត្យវត្តមានដោយដៃក្នុងម៉ោងសិក្សា។
- អ្នកគ្រប់គ្រងត្រូវប្រមូល និងផ្ទៀងផ្ទាត់ទិន្នន័យវត្តមានពីប្រភពជាច្រើន។
- និស្សិតដែលអវត្តមានជាញឹកញាប់អាចត្រូវបានរកឃើញយឺតពេល។
- ការបង្កើតរបាយការណ៍មានភាពយឺត និងត្រូវការការគណនាដោយដៃ។
- នាយកដ្ឋានសិក្សាខ្វះការមើលឃើញពេលវេលាជាក់ស្ដែងលើបញ្ហាវត្តមាន។

HRU-ATMS ដោះស្រាយបញ្ហាទាំងនេះដោយផ្ដល់ប្រព័ន្ធតែមួយដែលរួមបញ្ចូលវត្តមាន ទិន្នន័យសិក្សា អ្នកប្រើប្រាស់ របាយការណ៍ ការជូនដំណឹង និងការគ្រប់គ្រងរដ្ឋបាល។

## 3. សេចក្ដីថ្លែងបញ្ហា

ដំណើរការវត្តមានដោយដៃ ឬពាក់កណ្ដាលដោយដៃនាពេលបច្ចុប្បន្ន មានភាពមិនមានប្រសិទ្ធភាព និងពិបាកតាមដាននៅកម្រិតធំ។ វាអាចពន្យារពេលការសម្រេចចិត្តផ្នែកសិក្សា បន្ថយភាពត្រឹមត្រូវនៃទិន្នន័យ និងធ្វើឱ្យអ្នកគ្រប់គ្រងពិបាកកំណត់អត្តសញ្ញាណនិស្សិត ឬគ្រូបង្រៀនដែលត្រូវការការយកចិត្តទុកដាក់។

បញ្ហាសំខាន់គឺការខ្វះប្រព័ន្ធគ្រប់គ្រងវត្តមានដែលមានការប្រមូលផ្ដុំ មានសុវត្ថិភាព និងស្វ័យប្រវត្តិ ដែលអាច៖

- ផ្ទៀងផ្ទាត់វត្តមានបានយ៉ាងមានប្រសិទ្ធភាព។
- រក្សាទុកទិន្នន័យវត្តមានបានយ៉ាងអាចទុកចិត្តបាន។
- បែងចែកសិទ្ធិអ្នកប្រើប្រាស់តាមតួនាទី។
- បង្កើតរបាយការណ៍សិក្សាដែលមានប្រយោជន៍។
- ជូនដំណឹងដល់អ្នកគ្រប់គ្រងអំពីបញ្ហាសំខាន់ៗ។
- គាំទ្រលំហូរការងារវត្តមានទាំងនិស្សិត និងគ្រូបង្រៀន។

## 4. គោលបំណងគម្រោង

### គោលបំណងទូទៅ

អភិវឌ្ឍប្រព័ន្ធគ្រប់គ្រងវត្តមានសិក្សាដែលមានសុវត្ថិភាព មានការប្រមូលផ្ដុំ និងងាយស្រួលប្រើ សម្រាប់ HRU ដើម្បីធ្វើឱ្យប្រសើរឡើងនូវការតាមដានវត្តមាន ការរាយការណ៍ និងការសម្រេចចិត្តរដ្ឋបាល។

### គោលបំណងជាក់លាក់

- ជំនួសការត្រួតពិនិត្យវត្តមានដោយដៃ ដោយវត្តមានឌីជីថលផ្អែកលើ QR។
- អនុញ្ញាតឱ្យគ្រូបង្រៀនគ្រប់គ្រងវគ្គវត្តមាន និងតាមដានការចុះវត្តមានរបស់និស្សិត។
- អនុញ្ញាតឱ្យអ្នកគ្រប់គ្រងគ្រប់គ្រងនិស្សិត គ្រូបង្រៀន ថ្នាក់ មុខវិជ្ជា នាយកដ្ឋាន ជំនាញ ក្រុម និងការចាត់តាំងឆមាស។
- គាំទ្រវត្តមានគ្រូបង្រៀនតាមរយៈការចូល និងចេញដោយផ្អែកលើកាលវិភាគបង្រៀន។
- រកឃើញនិស្សិតដែលមានហានិភ័យ ឬស្ថិតក្នុងបញ្ជីខ្មៅដោយសារអវត្តមានច្រើនពេក។
- បង្កើតរបាយការណ៍ PDF និង Excel សម្រាប់និស្សិត មុខវិជ្ជា នាយកដ្ឋាន បញ្ហាវត្តមាន វត្តមានគ្រូបង្រៀន និងលទ្ធផលឆមាស។
- ផ្ដល់ការផ្ទៀងផ្ទាត់អត្តសញ្ញាណដែលមានសុវត្ថិភាព និងការគ្រប់គ្រងសិទ្ធិតាមតួនាទី។
- បញ្ចូលការជូនដំណឹងតាម Telegram សម្រាប់របាយការណ៍សំខាន់ៗ និងការជូនដំណឹងអំពីវត្តមាន។
- គាំទ្រការទំនាក់ទំនងពេលវេលាជាក់ស្ដែងរវាងអ្នកគ្រប់គ្រង និងគ្រូបង្រៀន។
- ផ្ដល់បរិស្ថានដាក់ប្រើប្រាស់ផ្អែកលើ Docker ដើម្បីងាយស្រួលដំឡើង និងថែទាំ។

## 5. វិសាលភាពគម្រោង

### វិសាលភាពដែលបានបញ្ចូល

គម្រោងនេះរួមបញ្ចូលម៉ូឌុលដូចខាងក្រោម៖

- ការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ និងការអនុម័តគណនី
- តួនាទីអ្នកគ្រប់គ្រងកំពូល អ្នកគ្រប់គ្រង គ្រូបង្រៀន និងនិស្សិត
- ការគ្រប់គ្រងនិស្សិត
- ការគ្រប់គ្រងគ្រូបង្រៀន និងគណនីគ្រូ
- ការគ្រប់គ្រងនាយកដ្ឋាន ជំនាញ ក្រុមថ្នាក់ មុខវិជ្ជា និងថ្នាក់
- វត្តមាននិស្សិតដោយប្រើការផ្ទៀងផ្ទាត់ QR code
- វត្តមានគ្រូបង្រៀនដោយផ្អែកលើកាលវិភាគបង្រៀន
- ការតាមដានវគ្គវត្តមាន
- ការវិភាគអវត្តមាននិស្សិត និងការតាមដានបញ្ជីខ្មៅ
- ការគ្រប់គ្រងពិន្ទុឆមាស និងលទ្ធផលសិក្សា
- ការនាំចេញរបាយការណ៍ជា PDF និង Excel
- ការបញ្ចូល Telegram bot
- API ជជែកពេលវេលាជាក់ស្ដែងសម្រាប់ការទំនាក់ទំនងរវាងអ្នកគ្រប់គ្រង និងគ្រូបង្រៀន
- ការកំណត់ប្រព័ន្ធ និងអត្តសញ្ញាណម៉ាក

## 10. ស្ថាបត្យកម្មប្រព័ន្ធ

```text
ចំណុចប្រទាក់អ្នកប្រើប្រាស់
    -> Web Routes / API Routes
    -> Controllers
    -> Form Requests
    -> Services
    -> Models and Repositories
    -> មូលដ្ឋានទិន្នន័យ MySQL
```

ការបញ្ចូលបន្ថែម៖

```text
Laravel Application
    -> Redis សម្រាប់ cache/queue
    -> Telegram Bot API សម្រាប់ការជូនដំណឹង
    -> Reverb សម្រាប់ chat/events ពេលវេលាជាក់ស្ដែង
    -> DomPDF និង Excel សម្រាប់របាយការណ៍
    -> Docker Compose សម្រាប់ការដាក់ប្រើប្រាស់
```
